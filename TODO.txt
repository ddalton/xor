Vision
=======
Support CRUD automation over REST using JPA
Allows developers to quickly add J2EE persistence to their UI development. Assists with RAD.
Helps bootstrap J2EE development and business logic can slowly be exposed to users over time.

1. Very fast object retrieval from an RDBMS using views, optimized for stored procedures
2. Native JSON support. Directly interact using JSON and hence just a JS library.
3. Support for models with loops using the swizzler JS library
4. Multiple query access methods:
   1. Default - By navigation
   2. OQL 
   3. SQL
   4. Stored procedure
5. Support Excel import/export for Perf data population
6. Shape support - type of versioning

Future enhancements
*******************
 

Support Amazon DynamoDB and Google cloud datastore
-- Aggregate is implemented using the following algorithm
-- Important fields are given their own columns (Support expanding this as needed, could be expensive)
-- All other data in the aggregate is stored as compressed JSON data in a separate column
By going with this approach we keep the API simple. Queries will work only with the columns identified as important fields

The compressed size limit is affected by the provider limits
DynamoDB - 400KB
Datastore - 1MB

The solution would be to split it across different rows. But that needs to be done by the application (Can be supported in the tool later).

-- Could consider supporting this approach in an RDBMS 
-- Support machine-learning performance improvement (Add frequently accessed columns and mark the row dirty if an important column is modified)
